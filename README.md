# Multithreaded File Management Daemon (Linux)

A professional-grade, multithreaded file management service designed for Linux. This system utilizes **IPC** [Inter-Process Communication] via System V Message Queues to coordinate file operations between independent client processes and a centralized background daemon.

## 🚀 Overview
Evolving from a kernel-level implementation (xv6), this project implements a **Client-Server Model** in user-space. It features a **Dynamic File Registry** that manages **Fine-Grained Locking** [resource-specific synchronization], allowing maximum concurrency without risking data corruption.

## 🏗️ Core Architecture
- **Event-Driven Daemon:** An infinite-loop background process that listens for `MsgPacket` structures on a message queue and dispatches **Worker Threads**.
- **Dynamic File Registry:** A linked-list-based in-memory structure that tracks active files. Each node contains a `pthread_rwlock_t` unique to that specific filename.
- **Reader-Writer Pattern:** - **Shared Access:** Multiple threads can perform `OP_READ`, `OP_META`, and `OP_COMPRESS` simultaneously.
    - **Exclusive Access:** `OP_WRITE`, `OP_DELETE`, `OP_RENAME`, and `OP_DECOMPRESS` require a **Writer Lock** [exclusive access] to prevent **Race Conditions**.

## 🛠️ Technical Stack
| Feature | Technology |
| :--- | :--- |
| **Communication** | System V Message Queues (`ftok`, `msgget`, `msgrcv`) |
| **Threading** | POSIX Threads (`libpthread`) with detached execution |
| **Synchronization** | `pthread_rwlock_t` (Per-file) and `pthread_mutex_t` (Audit Log) |
| **Compression** | System wrappers for `gzip` and `gunzip` |
| **I/O** | Linux System Calls (`open`, `fsync`, `stat`, `unlink`, `rename`) |

## 📂 Project Structure
- `filemanager.c`: The core Daemon (Server) containing the registry logic and worker functions.
- `testfile.c`: Basic functional test suite for core CRUD operations.
- `testfile2.c`: Comprehensive test suite including **Compression**, **Decompression**, and **Stress Testing**.
- `system.log`: The **Audit Trail** [chronological security record] generated by the system.
